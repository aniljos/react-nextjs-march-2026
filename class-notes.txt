React Components

Types of components(based on coding): Functional and Class
Types of components( based on rendering): client-side rendering(CSR) and server-side rendering(SSR)

In Nextjs: default rendering is SSR

-React Component props
	- parameterize a component
	- passing data from parent to child and vice-versa
	- changes to props will re-render the component
	- props is read-only
React Component state
	- internal to the component
	- changes to state will re-render the component
	- state changes are asynchronous
	- state changes are batches
	- setState
		- setState(updatedValue)
		- setState(callback)   (updatedValue callback(prevValue))
React effects(useEffect)
	- Hook into the lifecycle
	- mount: 
		- called once the component is mounted
		- dependency array is empty
		- API calls 
		- DOM initialization
		- Register to any events(document/window)
	-unmount
		- called once when the component is unmounted
		- cancel the api calls
		- Unregister to any events(document/window)
	- update
		- called when the component is rerendered
		


input element
	- binding using state(useState) => value, onChange : Controlled input
	- Reference(useRef) to the input and read the value : Uncontrolled input

Navigation in React	
	- There no apis that React provides for Navigation
	- react-router-dom: 
		- library for navigation
		- Create-React-App or Vite 
	- Next.js has a nvagiation library(The App and Page router)
				
React Design & Optimizations
	- Components are granular( small components)
	- Custom Hooks: reuseable, function thats uses existing hooks.
	- memoization: 
		- like caching 
		- React.memo(component)
		- The component re-renders only if the props or state changes

Session State(in the browser)
	- LocalStorage & SessionStorage
	- Cookies
	- Memory
	
Redux
	- Action is an object with a type & a payload
		- {type: "save_token", token: "abc"}
		- {type: "clear_token"}
	- Reducer is a function 
		- receives the actions
		- updates the store
		- always synchronous(cannot make an api call or other async tasks)
React-redux
	- Provider: component to integrate the store with the react application
	- Hooks	
		- useDispatch: returns the dispatch method
		- useStore: returns a reference to the store
		- useSelector: return a subset(of the state) in redux and subscribe to the store
Redux Tooklit
	- implementations of redux patterns
	- opinionted framework
	- slices => alternative to reducers
		- treat state as mutable
		- generates the reducer code
		- geneartes the action creators(actions creators are helper function to create an action)
